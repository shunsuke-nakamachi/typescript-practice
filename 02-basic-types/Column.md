# Column: Chapter 02 の補足

## 詰まりそうなポイント：`any` の誘惑
「型エラーが消えないからとりあえず `any` にしちゃおう」というのは初心者が一番陥りやすい罠です。`any` を使うと TypeScript はその変数のチェックを諦めてしまいます。可能な限り `unknown` を使うか、型を正しく定義する癖をつけましょう。

## 豆知識：`void` と `undefined`
関数が何も返さないとき、TS では `void` 型を使います。一方で、JavaScript の仕様として明示的に return しない関数は `undefined` を返します。
「型としての意味」は `void`（値を無視する）、「値としての実態」は `undefined`（未定義）という違いがあります。
