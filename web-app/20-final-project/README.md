# Chapter 20: 最終プロジェクト - Bookstore API

これまでの知識を全て組み合わせて、実用的な「本屋管理 API」を構築します。

## プロジェクト要件

1. **CRUD 操作**: 本（Book）の登録、一覧取得、詳細取得、更新、削除。
2. **バリデーション**: Zod を使用して、入力データの形式を厳密にチェック。
3. **OpenAPI**: Swagger UI で操作可能なAPIドキュメントを自動生成。
4. **認証**: 特定の更新（登録・更新・削除）には、ベーシック認証または JWT が必要。
5. **テスト**: Vitest を使用して、主要なエンドポイントのテストを書く。

## ヒント

- `db.ts` で本のデータを管理するクラスを作りましょう。
- `app.ts` で Hono インスタンスを作成し、ミドルウェアとルートを設定します。
- `index.ts` でサーバー（node-server）を起動します。
