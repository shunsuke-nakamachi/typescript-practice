# Chapter 29: API デザインの原則 (API Design Principles)

「使える API」から「使いやすい API」へステップアップしましょう。

## 1. RESTful ベストプラクティス

- **名詞を使う**: `/getUsers` ではなく `/users`。
- **階層構造**: `/users/1/posts`（ユーザー1の投稿一覧）。
- **バージョニング**: `/v1/users`。将来大きな変更をするときに壊さないためです。

## 2. ページネーションとフィルタリング

データが1万件あるとき、一度に全て返すのは危険です。
`/users?page=1&limit=20` のようなパラメータを設計に取り入れましょう。

## 演習
